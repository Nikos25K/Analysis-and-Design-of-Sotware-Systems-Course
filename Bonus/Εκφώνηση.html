Περιγραφή:

Καλείστε να επεκτείνετε το σύστημα που σχεδιάσατε στην υποχρεωτική εργασία προσθέτοντας επιπλέον οντότητες, σύμφωνα με τις απαιτήσεις του χρήστη. Πιο συγκεκριμένα:

  • Θα πρέπει να πάρετε συνέντευξη από τον χρήστη προκειμένου να συγκεντρώσετε τις απαιτήσεις. Η διάρκεια της συνέντευξης δεν θα πρέπει να υπερβαίνει τα 2'.
  • Η συνέντευξη θα πρέπει να μετατραπεί σε κείμενο.
  • Με βάση τις απαιτήσεις θα προκύψει μια νέα Αντικειμενοστραφής Ανάλυση/Σχεδίαση, η οποία θα αποτελεί επέκταση της υποχρεωτικής εργασίας σας.
  • Θα ακολουθήσει η υλοποίηση της σχεδίασης με κάποια γλώσσα προγραμματισμού, η οποία υποστηρίζεται από το Github Actions για build, test και deploy. Ενδεικτικά, προτείνεται η Python.
  • Στη συνέχεια θα πρέπει να γίνει ο έλεγχος του κώδικα που προέκυψε (unit testing) μέσω Github Actions.

Οδηγίες:

  1. Η (bonus) εργασία μπορεί να γίνει από 1 ή 2 άτομα. Σε περίπτωση 2 ατόμων, δεν είναι δεσμευτικό να είναι η ίδια δυάδα με την υποχρεωτική εργασία.
  2. Η εργασία δίνει έως μία μονάδα στον τελικό βαθμό του μαθήματος. Απαραίτηση προϋπόθεση είναι ο φοιτητής να έχει βαθμό τουλάχιστον 5 στην υποχρεωτική εργασία.
  3. Κάθε εργασία θα είναι σε ιδιωτικό αποθετήριο στο Github, στο οποίο θα πρέπει να προσθέσετε ως collaborator τoν διδάσκοντα (diogenisAl). Η δομή του αποθετηρίου θα πρέπει να είναι η ίδια με το Αποθετήριο-Δείγμα.
  4. Στην περίπτωση της δυάδας, η δουλειά του κάθε μέλους θα πρέπει να αποτυπώνεται στο ιστορικό των commits του αποθετηρίου.
  5. Κάθε φοιτητής (ή δυάδα) θα πρέπει να συντάξει μια αναφορά (pdf) στην οποία θα αναφέρονται: όνομα, επίθετο, Α.Μ., e-mail, σύνδεσμος Github αποθετηρίου με την εργασία και ένα σύντομο κείμενο (έως 200 λέξεις) με την περιγραφή της εργασίας που έγινε. Η αναφορά θα υποβληθεί στο eclass.
  6. Η τελική ημερομηνία υποβολής της αναφοράς στο eclass (περιοχή: Εργασίες) είναι την Κυριακή, 30 Ιουνίου 2024 - 11:55 μ.μ. και δεν προβλέπεται να δοθεί παράταση. Μετά από αυτήν την ημερομηνία δεν θα πρέπει να γίνει οποιαδήποτε αλλαγή στο αποθετήριο, μέχρι να ανακοινωθούν οι βαθμοί.

 

Προδιαγραφές εργασίας:

  1. Στον φάκελο 01_Requirements του αποθετηρίου σας θα πρέπει να είναι το κείμενο της συνέντευξης. Τα στοιχεία του ατόμου που έδωσε τη συνέντευξη θα είναι κρυφά.
  2. Στον φάκελο 02_Design του αποθετηρίου θα πρέπει να υπάρχουν 2 αρχεία pdf με διαγράμματα: (1) το αρχείο pdf με το διάγραμμα κλάσεων της επέκτασης που δημιουργήσατε (τουλάχιστον 2 κλάσεις για ατομική εργασία, τουλάχιστον 4 κλάσεις για ομαδική εργασία) και (2) το αρχείο pdf με ένα διάγραμμα της επιλογής σας.
  3. Στον φάκελο 03_Code του αποθετηρίου θα πρέπει να υπάρχουν τα αρχεία κώδικα με τις κλάσεις που προσθέσατε (υλοποίηση όλων των attributes & methods), καθώς και το αρχείο με το unit testing (τουλάχιστον 2 έλεγχοι για κάθε κλάση).
  4.  Στο Github Actions αποτυπώνεται εάν οι έλεγχοι (unit testing) είναι επιτυχείς ή όχι. Για τον πλήρη βαθμό, όλοι οι έλεγχοι θα πρέπει να είναι επιτυχείς.
